# 文件大小计算器 - 规格文档

## 1. 项目概述

- **项目名称**: URL文件大小计算器
- **项目类型**: Python桌面应用 (Tkinter)
- **核心功能**: 读取Excel中的URL列表，获取每个文件的大小，汇总总量
- **目标用户**: 需要处理大量文件下载链接的用户

## 2. UI/UX 规格

### 布局结构

- **窗口尺寸**: 900x650 像素
- **布局**: 垂直布局
  - 顶部：工具栏（选择文件按钮 + 文件路径显示）
  - 中部：表格（显示URL和文件大小）
  - 底部：状态栏（总量统计 + 进度信息）

### 视觉设计

- **配色方案**:
  - 主色: #2563EB (蓝色)
  - 背景: #F8FAFC
  - 文字: #1E293B
  - 边框: #E2E8F0
  - 悬停: #DBEAFE

- **字体**: 
  - 主字体: Segoe UI, 10px
  - 表头: 12px, 加粗
  - 数字: Consolas (等宽)

### 组件

1. **工具栏**
   - "选择Excel文件" 按钮 (蓝色背景，白色文字)
   - 文件路径标签 (灰色背景)

2. **结果表格**
   - 列: 序号 | URL | 文件大小 | 状态
   - 行高: 32px
   - 斑马纹背景
   - 状态图标: ✓ 成功, ✗ 失败

3. **状态栏**
   - 文件总数
   - 成功获取数量
   - 总大小 (格式化显示: KB/MB/GB)
   - 进度条 (处理中显示)

## 3. 功能规格

### 核心功能

1. **Excel读取**
   - 支持 .xlsx 和 .xls 格式
   - 自动查找第一个工作表
   - 第一列应为URL列
   - 允许空单元格自动跳过

2. **URL处理**
   - 发送HTTP HEAD请求获取Content-Length
   - 支持HTTP和HTTPS
   - 超时时间: 10秒
   - 失败重试: 1次

3. **结果展示**
   - 实时更新表格
   - 文件大小格式化 (B/KB/MB/GB)
   - 状态显示 (成功/失败/超时)

4. **总量计算**
   - 所有成功获取的大小累加
   - 格式化显示

### 交互流程

1. 用户点击"选择Excel文件"
2. 系统打开文件选择对话框
3. 用户选择Excel文件
4. 系统自动开始处理URL
5. 表格逐步显示结果
6. 处理完成后显示总量
